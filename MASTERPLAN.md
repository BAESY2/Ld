# MASTERPLAN — 자동화 제어합성의 신뢰 난제를 부수는 초대형 계획

> 정직이 유일한 해자다. 이 문서는 과장하지 않는다. 못 하는 건 못 한다고 적는다.

## 0. 부수려는 난제 (정확히)
**전문가 희소성 × 물리적 위험.** 산업 제어 로직(PLC)을 짤 수 있는 사람은 희소하고,
틀린 로직은 설비파손·인명사고다. 그래서 LLM 환각으로는 *안전하게* 자동화할 수 없다.
→ 난제 = **의도(자연어)로부터 제어 로직을 *신뢰 가능하게* 자율 합성**. 여기서 '신뢰
가능'은 "아마 맞음"이 아니라 **형식 증명 + 못하면 정직 거절**을 뜻한다.

## 1. 중심 베팅 (왜 우리가 시간이 갈수록 이긴다)
**검증기가 해자다.** 생성기는 모두 좋아진다(템플릿→우리 한국어엔진→LLM). 방어 가능한
희소 자산은 *무엇이 생성되든* 안전을 기계증명하고, 못 고치면 거절하며, 반례로 수리를
이끄는 **빠르고 일반적인 검증+수리 루프**다. 모델이 강해질수록 "안전 게이트"의 가치는
*커진다*. 능력과 안전을 동급으로 — 단 안전은 형식증명으로(RLHF 희망이 아니라).

세 기둥:
- **이해(Understand)**: AI 없이 한국어 의도를 *구조적·설명가능*하게 파싱(형태소→프레임).
- **합성(Synthesize)**: 결정론 코어(템플릿/스캐폴드) + (키 시)LLM, 둘 다 게이트 통과해야 채택.
- **보장(Guarantee)**: 건전한 형식검증(인터락·타이밍·도달성) + 반례 수리 + 정직 거절. ← 해자.

## 2. 둘이서, 키 없이, 세계급으로 만들 수 있는 것 (현실 경계)
- ✅ **세계에서 가장 엄밀한 오픈 제어로직 검증기 + 합성루프** — 키 불필요, 팀 불필요, 측정 가능.
- ✅ **AI 없는 한국어 제어 이해 엔진** — 차별적·결정론·한국시장.
- ✅ **재현 가능한 벤치마크/건전성 실증** — 논문급 데이터.
- ❌ 안전 인증(TÜV/SIL)·실하드웨어 양산·시장유통 — 조직 필요. 숨기지 않는다.

## 3. 북극성 (측정·회귀 금지)
- 검증기 건전성@규모: false-proof=0, miss=0 (현재 406쌍 → 수천, 속성 다종).
- 한국어 이해: 프레임추출 정확도·확신 보정(calibration), BM25 대비 우월 *수치*.
- 침묵실패율(자신있게 틀림): realworld ≤1%, in-template 누출 0% (불변).
- 검증 *대상 클래스*: 인터락 → +타이밍 → +아날로그 의미론.

## 4. 단계 (각 단계 = 측정 가능한 한 방, 멈추지 않고 직진)
- **M1 이해**: korean.py 형태소 → `app/intent.py` 절·프레임(조건→동작·대상·극성·수량) →
  레시피 매핑 + 파스커버리지 기반 설명가능 확신도 → **BM25 ablation 벤치**(발명 증명).
- **M2 보장-확대**: 타이밍(타이머 의미론) k-귀납 + 실장비(OpenPLC) 오라클 건전성 재검증.
- **M3 합성-일반화**: 프레임→다중모듈/교차인터락 자동 합성(키 없이 커버리지 확대),
  (키 시) LLM 생성→게이트 lift 측정(raw-LLM 침묵실패 대비).
- **M4 검증기 일반화**: 임의 ST(우리 합성기 밖)도 검증·반례·수리하는 일반 게이트 = 해자 완성.
- **M5 증거**: 한 시나리오 end-to-end(한국어→검증래더→OpenPLC 트레이스 일치) 재현 패키지.

## 5. 작업 규율 (불변 게이트)
구현→같은 커밋 pytest→ruff/mypy/pytest+벤치→회귀0→커밋·푸시. 측정 없는 주장 금지.
하드게이트: in-template 정확도·21개밖 침묵0%·이중코일0·인터락0·골든100%·건전성(false-proof=0).

## 5b. 목적 (완료 기준 — 이게 충족되면 '대규모 컴파일러'가 선다)
레시피(37) 대신 **결정론 한국어→검증 제어 컴파일러**가 *주(主) 엔진*이 되고 넓게 표현한다.
- **G1 어휘 폭**: 기기·조건유형·동작 ~3배↑ → 생성성 벤치 distinct ≫ 1525, 현장 표현 커버 확대.
- **G2 컴파일러 의미**: 인터락 자동추론(동시금지/정역), 엣지·원샷, 시퀀스/타이밍 조합.
- **G3 제품 연결**: 컴파일러가 API(/api/compile)·스튜디오의 *기본 합성 경로*(레시피는 폴백).
- **G4 정직 측정**: 대규모 NL→컴파일→검증 벤치(커버리지%·침묵실패0) + 생성성 수치.
- **G5 게이트 불변**: 1000+ pytest·false-proof0·이중코일0·in-template84%·골든100% 전구간.
충족 전까지 병렬 라운드를 멈추지 않는다(사용자 확인 없이 진행).

## 5c. 목적 달성 현황 (자율 라운드 1~3)

- **G4 정직 측정 ✓**: compile_bench 52건 — easy/compound/quantified 100% confident·verify·

- **G5 게이트 ✓**: 1096 passed·골든21/21·이중코일0·인터락0·false-proof0 전구간.

## 6. 진척 (병렬 실행)
- **M1 이해 ✓**: 형태소(korean) → 의도프레임(intent) → 구조 레시피매핑 → 무편향 벤치.
  결과: 띄어쓰기 제거 교란에서 문법엔진 견고(커버 의도 100% 유지)·BM25 90%→80%, 어휘형은
  정직 기권(거짓매핑 0) = 상보 하이브리드.
- **M2 보장확대 ✓**: 그룹 상호배제(one-hot/at-most-one) k-귀납 추가. 건전성@규모 1000개로
  확대 — 쌍 2186·그룹 397 모두 false-proof=0, miss=0, detection 100%.
- **M5 증거 ✓(1차)**: end-to-end 데모(scripts/demo_e2e.py, docs/DEMO.md) — 한국어 한 문장이
  이해→검증→래더까지 전사로, 비확신/도메인밖은 보류(환각 0).
- **다음**: M3 합성-일반화(프레임→다중모듈/교차인터락 자동합성), M4 검증기 일반화(임의 ST),
  무편향 벤치 대규모화, M2 타이밍 의미론.

## 7. M4 캡스톤 (Round 5 — 검증기가 해자)
- **자동 수리(app/repair.py)**: 검증 실패 ST 를 건전하게 수리→재검증. 이중코일→M릴레이 OR
  병합(의미보존), 인터락 위반→가드 'AND NOT 상대' 주입(안전측·멱등), 구조적 결함→정직 거절.
  종료성 보장. 수리 효능 벤치: 고장 12건 전부 수리해 재검증 통과(100%, 잔존실패 0).
- **export 라운드트립**: 컴파일 산출(시퀀서·다중인스턴스·비교기)의 PLCopen XML·벤더 IL
  내보내기 깨짐 2건 수정(비교기 변수 미선언→IDE 임포트, IL 비교접점 연산자 붙음).

## 9. Round 7 — 의미타입 가드 + 통합 스코어카드

- **아날로그 수치시뮬 검증**: 히스테리시스 밴드(3바↓ON/5바↑OFF, 4바=방향의존 유지) 실동 입증.
