# ROADMAP — "정직한 프로토타입"에서 "펀딩 가능한 제품"으로

## 투자 thesis (난제 = 해자)
**푸는 난제:** 자연어 → *틀리지 않는* 배포 가능한 PLC 제어 코드. (LD 자유생성은 미해결 난제 —
LLM 은 래더를 환각하고, 산업 제어에서 틀린 코드는 설비파손·인명사고다.)
**해자:** ① 정형 검증 게이트(Z3 인터락 증명 + 이중코일 구조적 제거 + 결정론 합성) — 출력이
"아마 맞음"이 아니라 핵심 안전속성에 대해 *수학적으로 증명됨*. ② **0% 침묵실패** — 못 하는 건
자신있게 틀리는 대신 거절·골격제시(안전 도메인의 헤드라인). ③ 벤더 중립 + 국내(LS/메카피온) 웨지.
**킬러 데모(만들 것):** 한국어 한 문장 → 검증 래더 + 안전성 증명서 + 실 OpenPLC 트레이스=시뮬 일치(라이브).
**펀딩 가능성의 갭:** 표현 범위(아날로그/PID/모션) · 재현가능 지표 대시보드 · 실장비 라운드트립 증명.

## 난제의 두 반쪽 (냉정한 분해) + 논문급 바
- **H1 커버리지**: 임의의 어려운 산업 의도(아날로그 동역학·PID·다축 모션·대규모) 생성. → 일반
  합성/LLM 필요. 키-게이트·거대. 부분 착수(아날로그 비교기).
- **H2 보장**: 무엇이 생성되든 *기계검증된 안전*. → 이 레포의 핵심·키 없이 논문급 가능.
  사용자(현장 비전문가)를 실제로 구하는 건 H2 — 위험 로직을 *증명 가능하게* 절대 안 내보냄.

**논문급 실증 (달성):** 검증기 건전성@규모 — 무작위 300+ 프로그램 + 실레시피 + 적대 뮤턴트의
**406개 인터락 쌍**에서, k-귀납이 *증명한* 171쌍을 실행오라클(디지털 트윈)이 단 한 번도 반증
못함(**false-proof=0**), 오라클이 찾은 232 위반을 형식검증이 전부 탐지(**miss=0**) →
양방향 건전성의 규모 실증(`scripts/soundness_study.py`, 회귀가드 `test_soundness_study.py`).
다음: 검증 *대상 클래스* 확대(타이밍/아날로그 의미론) + 외부 벤치마크 + (키 시)LLM 합성 lift.

## 0. 정직한 스코프 (무엇을 목표로 하고, 무엇은 아닌가)

지멘스(TIA Portal + 인증 F-CPU)는 **인증 하드웨어 + 수십 년 런타임을 가진 회사의 제품**이다.
혼자, 키 없이, 이 레포에서 그것과 *기능안전 인증*을 겨룰 수는 없다 — 그건 조직·시험기관·
하드웨어가 필요한 영역이고, 본 로드맵은 그 점을 숨기지 않는다.

**달성 목표(현실적 북극성):** "비전문가의 한국어 요구 → *정형 검증된* 멀티벤더 래더/ST를,
*못 하는 것은 정직하게 거절하며*, 실장비까지 라운드트립으로 검증해 내보내는 **신뢰 가능한
생성·검증 백엔드**." 즉 지멘스 *대체*가 아니라, LS/지멘스 툴체인에 먹여주는 **프런트엔드
가속기 + 정형 안전망**의 완성형.

## 2. 단계 (Phase) — 각 단계는 독립 빌드·검증·커밋, 병렬화 가능

### P1 — 아날로그 비교 코어 (가장 큰 "범위" 갭) ★최우선
키 없이 표현 불가였던 *아날로그 설정값* 클래스를 IEC 표준으로 흡수한다.
- **W1.1** 모델: `Comparator`(BOOL 플래그 ← 아날로그 신호 `op` 임계, 선택 히스테리시스).
  REAL/INT 입력 + GT/GE/LT/LE/EQ. Z3 는 플래그를 원자로 취급(건전·보수적).
- **W1.2** synth: 비교식 ST 방출(`FLAG := SIG >= THR;`), 히스테리시스는 SR 래치.
- **W1.3** verifier/transpiler: 플래그 단일구동 보장, 래더 접점 매핑.
- **W1.4** 레시피 2종: `pressure_band`(히스테리시스 압력), `temp_setpoint`(승온/유지/냉각 with 목표온도). nlmatch: 아날로그 설정값을 *거절*→*표현*으로 라우팅.
- **완료 기준:** quant-03/07 류가 검증 통과 산출. nl_bench·0% 침묵 유지. 골든+신규.

### P2 — 측정 정직성 & compound 캡처
- **W2.1** realworld 벤치가 scaffold/compose의 *전체의도 캡처*를 실측(현재 0% 하드코딩 제거).
- **W2.2** compound conf_partial 3건(compound-03/06/12) scaffold 자동확장.
- **W2.3** watchdog/timeout 레시피(compound-11의 "X초 내 무응답→알람").
- **완료 기준:** compound 전체의도 캡처 ≥ 40%, 침묵실패 ≤ 1%.

### P3 — 결정론 렌더러 완성 (LLM 의존 축소, "규모" 토대)
- **W3.1** ST→래더 트랜스파일러 커버리지 확대(CASE/중첩식), 골든 ST 라운드트립.
- **W3.2** 다중 중첩 브랜치(재귀 LadderNode) — 복합 인터락/병렬.
- **완료 기준:** 골든 ST N개 → 기대 래더 트리 일치, 이중코일 0.

### P4 — IDE 라운드트립 & 실장비 (상호운용 "라스트마일")
- **W4.1** PLCopen 그래픽 LD export(ST 본문 넘어 접점/코일 XML) → OpenPLC Editor 임포트.
- **W4.2** FEnet/OpenPLC live 적합성 회귀(env-gated), 라운드트립 차이 0.
- **완료 기준:** 내보낸 XML이 외부 IDE에서 열리고, 실장비 트레이스 = 시뮬 트레이스.

### P5 — 검증 깊이 (안전 신뢰 강화, 인증은 불가하나 *증명*은 깊게)
- **W5.1** k-induction 불변식 확대(상호배제·안전상태 유지 증명).
- **W5.2** 안전패턴 라이브러리(2-hand/가드/뮤팅) 정형 속성 + 메타모픽 테스트.
- **완료 기준:** 안전 레시피마다 정형 속성 1+개 증명, 반례 회귀 가드.

### P6 — 하드닝 & 규모
- 대형 프로젝트(수십 모듈) 합성 성능·주소충돌 0 스트레스, API 견고화.

## 3. 실행 프로토콜 (멈추지 않는 루프)
