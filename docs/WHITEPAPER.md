# 백서 — 자연어로부터 *형식 검증된* PLC 제어를 합성하다

**부제: 결정론 한국어 의도 컴파일러 + 기계증명 안전 게이트**
버전 1.0 · 연구 프리뷰 · 모든 수치는 `python scripts/capability_report.py` 로 재현된다.

---

## 초록 (Abstract)

산업 제어 로직(PLC 래더)을 작성할 수 있는 엔지니어는 희소하고, 잘못된 로직은 설비
파손·인명 사고로 직결된다. 대규모 언어모델(LLM)은 그럴듯한 코드를 생성하지만 *환각*
하므로, 검증 없이 산업 현장에 투입할 수 없다. 본 연구는 다른 길을 택한다: **(1)** 한국어
제어 지시를 *결정론* 형태소·구문 엔진으로 구조 분석하고(LLM·API 키 불필요), **(2)**
의도 프레임을 IEC 61131-3 Structured Text 로 *컴파일*하며, **(3)** 산출물을 SMT(Z3)
기반 k-귀납으로 **기계 증명**하고 못 고치면 *정직하게 거절*한다. 핵심 주장은 "잘 생성
한다"가 아니라 **"모르는 것은 만들지 않는다(침묵 실패 0)"** 이며, 이를 적대 벤치마크
110건·컴파일 코퍼스 52건에서 **침묵 실패 0**으로, 검증기 건전성에서 **false-proof 0 ·
miss 0**으로 측정한다.

---

## 1. 풀려는 난제

> **전문가 희소성 × 물리적 위험.**

- 제어 엔지니어는 양성에 수년이 걸리고, 중소 제조 현장일수록 부족하다.
- 래더 한 줄의 오류(예: 동일 출력 이중 코일, 정·역 동시 투입, 스타-델타 동시 투입)는
  곧 모터 소손·상간 단락·기계 충돌이다.
- 따라서 "자연어→즉시 기계"는 **정확함을 증명할 수 없으면 무의미**하다. 이것이 본
  프로젝트가 *생성*이 아니라 ***검증***을 1순위로 두는 이유다.

---

## 2. 중심 베팅 — "검증기가 해자다"

생성기는 시간이 지나면 누구나 좋아진다(템플릿 → 우리 한국어 엔진 → LLM). 방어
가능한 희소 자산은 *무엇이 생성되든* 안전을 기계증명하고, 못 고치면 거절하며, 반례로
수리를 이끄는 **빠르고 일반적인 검증+수리 루프**다. **모델이 강해질수록 이 안전
게이트의 가치는 커진다** — 능력과 안전을 동급으로 두되, 안전은 RLHF 희망이 아니라
*형식 증명*으로 보장한다.

세 기둥:

| 기둥 | 내용 | 키 필요 |
|---|---|---|
| **이해(Understand)** | 한글 자모 산술 → 조사/어미 → 의도 프레임(조건×동작×기기×수량) | ❌ |
| **합성(Synthesize)** | 프레임 → 검증 가능한 StateMachineSpec → ST → 래더 | ❌ |
| **보장(Guarantee)** | Z3 k-귀납 인터락/상호배제 증명 + 반례 수리 + 정직 거절 | ❌ |

   │
   └─► app/plant.py   설비 설계도(태그·BOM·PLC 주소·배관/신호 연결·전기 산식 선정)

- IEC 61131-3 표준 타입만(BOOL/INT/DINT/REAL/TIME/WORD).
- LS 디바이스 체계만(P/M/T/C/D).
- 동일 출력 이중 코일 금지(구조적 + 텍스트 검사).
- LLM 호출은 전 테스트에서 mock — **API 키 없이 CI 통과**.

## 5. 형식 보장의 구체 사례 — 표준 기동 회로

본 컴파일러는 한국어 한 줄에서 현업 표준 회로를 합성하고, 그 *안전 속성을 증명*한다.

### 5.1 스타-델타(Y-Δ) 기동 — "5.5킬로와트 모터 스타델타로 기동해"
```
MOTOR   := (START OR MOTOR) AND NOT (STOP);          ' 주접촉기 자기유지
T1(IN := MOTOR, PT := T#7s);                          ' 전환 타이머
MOTOR_D := (MOTOR AND T1.Q) AND NOT MOTOR_Y;          ' Δ접촉기(선평가→개방전환)
MOTOR_Y := MOTOR AND NOT T1.Q AND NOT MOTOR_D;        ' Y접촉기
```
- **증명**: `MOTOR_Y ⊥ MOTOR_D` 동시 투입(상간 단락) 불가가 k-귀납으로 증명됨.
- **시뮬 실측**: 전환 시 동시 ON **0 스캔**, 개방전환 데드타임 ≥1 스캔.
- **전기 산식 자동 선정**(380V IE3 전류표 + `I=P/(√3·V·cosφ·η)`):
  FLA 11.8A → **MCCB 3P 20AT · MC-12 · EOCR 설정 11.8A**.

### 5.2 비상정지(E-STOP) — 전 출력 강제 차단
`… 비상정지 누르면 다 꺼` → 컴파일러가 **모든 출력 식에 `AND NOT ESTOP` 가드를 박는다**.
구조적으로 ESTOP=1 인 어떤 스캔에서도 출력이 1 일 수 없으며, 디지털 트윈에서 전 입력을
켜도(ESTOP 포함) 출력 전부 0 임을 실측 확인한다(안전 기본동작 by construction).

### 5.3 컨베이어 캐스케이드(연동) — 새 증명 클래스 "포함관계"
`컨베이어 3대 연동해` → 하류부터 기동(2s 간격), 상류 식은 하류 식의 *강화*(누적 조건
인라인). 안전속성은 상호배제가 아니라 **포함관계 □(상류→하류)** — "상류가 돌면 하류는
반드시 돌고 있다"(자재 적체/낙하 방지) — 를 k-귀납으로 증명한다
(`verifier.prove_containment`). 개발 중 증명기가 하류-심볼-가드 회로의 *1스캔 갭*
결함을 잡아내 회로를 교정했다 — 검증기가 설계를 바로잡은 실사례.

### 5.4 정역 운전 / 5.5 펌프 교번
- 정역: `MOTOR_FWD ⊥ MOTOR_REV` 증명(역상 단락 방지).
- 교번: 기동 엣지마다 1호기/2호기 토글, `PUMP1 ⊥ PUMP2` 증명 + 시뮬로 교번 실거동
  (1회차 1호기 → 재기동 2호기 → 동시 ON 0) 검증.

## 6. 정직한 한계 (숨기지 않는다)

- **트랙션 0** — 사용자·매출·파일럿 없음. 연구 프리뷰다.
- **표현 범위가 좁다** — 불리언/시퀀스/기초 아날로그 비교·표준 기동 회로까지.
  **PID·모션/서보·통신(Modbus)·HMI·대규모 분산제어는 거절**(못 함).
- **다중 서브시스템 *전체의도* 자동합성은 구조적 0%** — 단일 레시피 한계(개별 합성은 됨).
- **검증은 인터락·시퀀스 상호배제 중심** — 타이밍 정량·아날로그 동역학 완전성은 미완.
- **자체+교차검증** — 외부 독립 감사·실하드웨어 양산·안전 인증(TÜV/SIL)은 범위 밖.

## 8. 결론

본 연구는 "AI가 코드를 잘 쓴다"가 아니라 **"AI가 무엇을 쓰든 안전을 기계증명하고,
못 하면 정직하게 거절한다"** 를 측정으로 증명하는 안전 게이트의 프로토타입이다.
침묵 실패 0 · false-proof 0 은 산업 자동화에서 LLM 단독 생성이 넘지 못하는 선이며,
이것이 본 프로젝트의 방어 가능한 가치다.
