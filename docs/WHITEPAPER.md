# 백서 — 디지털트윈 데모 매뉴얼·산식 명세

> 대상: `docs/demo/index.html` (9라인 라이브 디지털트윈) · 기준일 2026-06-11
> 모든 수치는 시뮬레이션 실측값이며, 아래 산식 그대로 계산된다(코드와 1:1).

## 1. 가상 PLC — 스캔 시맨틱

매 프레임(Δt, rAF 기준 ≈16.7ms / 서버 코사임 10ms 고정) 래더 JSON을 위→아래 순차 평가:

1. **입력 래치**: 버튼·센서 값이 변수 테이블에 기록
2. **렁 조건**: `rung = OR(브랜치)`, `브랜치 = AND(접점)` — NO 접점=값, NC 접점=¬값
3. **출력 기록**:
   - COIL: `out := rung`
   - TON 타이머: `IN=rung` 일 때 `acc := min(acc+Δt, PT)`, `Q := (acc ≥ PT)`; IN 하강 시 `acc:=0, Q:=false` (상승엣지 스캔은 acc=0부터 — 실PLC 동일)
   - CTU 카운터: rung 상승엣지마다 `cnt := cnt+1`, `Q := (cnt ≥ PV)`; `R :=` 조건 참이면 `cnt:=0`
   - SET/RESET 코일: 조건 참일 때 래치/해제
4. 자기유지(seal-in)는 직전 스캔 출력값을 접점으로 참조 — 스캔 순차성에 의해 결정론

서버 코사임(`/api/cosim`)은 동일 시맨틱을 OpenPLC 런타임과 bit-for-bit 차분검증된
코어(`app/simulator.py`)로 수행하며, 전 스캔 I/O 트레이스를 기록한다(상한 60,000스캔).

## 2. 생산 KPI 산식

| 지표 | 산식 | 비고 |
|---|---|---|
| 택타임 t_takt | `Δt_완성 / Δn` (완성 이벤트 간 경과시간/개수) | EMA 평활: `takt ← 0.7·takt + 0.3·신규` |
| UPH | `3600 / takt` | 시간당 생산 환산 |
| 누적 중량 | `생산수량 × 개당 중량(kg)` | 제품 파라미터표 §6 |
| 재공 WIP | 라인 위 미완성 수량(씬 물리 카운트) | |
| 라인 속도 | 벨트 속도 m/s — 가감속 램프 `v ← v ± a·Δt` (a=1.2~2.5 m/s²) | 속도 슬라이더 = 목표속도 ×50~150% |
| 가동률 A | `주설비 누적 RUN시간 / 경과시간` | 상시가동 라인(검수·비전·펌프장)은 A=1 고정 |
| 불량율 | `NG / (OK + NG)` | 비전 라인: 유출(미검사 NG의 양품 적재)은 품질에서 차감 |

## 3. OEE

```
OEE = A × P × Q
  A (가용률)  = 주설비 RUN시간 / 경과시간            (상시가동 라인 = 1)
  P (성능)    = min(1, 목표 택타임 / 실측 택타임)     (실측 전 = 미표시)
  Q (품질)    = OK / (OK + NG)                       (비전 라인: (OK − 유출) / (OK + NG))
```
OEE < 55% 시 경고색. 택타임이 목표×1.3 초과 시 알람 이벤트 발생(×1.1 미만 복귀 시 해제).

## 4. 전력·부하 산식

| 항목 | 산식 |
|---|---|
| 기기 소비전력 | `P[kW] = √3 × 380V × I_정격[A] × PF(0.85) / 1000` — ON 기기 합산 |
| 누적 전력량 | `E[kWh] = Σ P·Δt / 3600` |
| 표시 전류 | `I = I_정격 × (0.93 + 0.05·sin(6t))` (RUN 시), 트립 시 0.0A |
| 부하율(모의) | `58 + 20·sin(0.5t) + 8·sin(2.2t)` % — 80% 초과 시 과부하 경보(기기당 20s 쿨다운) |

기기 정격전류는 운전 데이터 표의 기기 사양(예: M-101 3.6A, P-801A/B 28A, RB-401 14.2A).

## 5. 고장 주입 시맨틱

| 유형 | 효과 |
|---|---|
| 모터 트립 | 유효출력 `eff(sym)=false` — 래더는 ON 지령 유지, 설비만 정지(TRIP 표시·전류 0) |
| 센서 고착(ON) | 매 스캔 입력 강제 true — CTU는 엣지 카운트라 계수 정지 현상 재현 |
| 카메라 정지 | 검사 스킵 → 게이트 미전환 → NG 유출 감지·경보·Q 차감 |

펌프장: 리드 트립 → 수위 하락 → `HIGH_DEMAND`(수위<32%) → 래더 로직만으로 래그 펌프 자동 기동(페일오버).

## 7. 아날로그(수치) 처리 — 엔진 코어

- 비교식 `VAR op N` (op ∈ {<,>,<=,>=,=,<>}, N=정수) — `300 > LEVEL`은 `LEVEL < 300`으로 정규화
- 시뮬레이터: INT 입력 타임라인 허용, 비교는 `int(table[var]) op N`
- 정형 검증: 비교식을 `z3.Int`로 인코딩 — 예: `LEVEL<300 ∧ LEVEL≥700 = UNSAT`(범위 배타 증명)
- 래더: 비교 접점 생성(`LEVEL < 300` NO 접점), `NOT(≥700)`은 NNF에서 `<700`으로 연산자 반전
- 트윈 계측 D 레지스터(펌프장 D0000 수위%·D0001 유량, 배치 D0000 레벨·D0001 드럼)는
  씬 물리값의 표시이며 PLC 메모리가 아님(명시 라벨)

## 8. 검증 항목 정의

- **이중 코일 0**: 동일 출력 심볼에 코일 대입 2회 이상 없음(스캔 결정론 보장)
- **Z3 인터록 UNSAT**: 배타 출력쌍(예: 정/역)의 동시 ON 조건식이 충족 불가능함을 SMT로 증명
- **도달성**: 각 상태(기동·유지·정지 등)에 도달하는 입력 시퀀스 존재 확인
- **OpenPLC 차분검증**: 동일 ST·입력 타임라인에 대해 시뮬레이터와 실런타임 출력 bit-for-bit 일치

## 9. 화면 조작 매뉴얼

| 조작 | 방법 |
|---|---|
| 라인 선택 | 상단 오버뷰 카드 클릭(전 라인 백그라운드 동시 가동·상태 유지) |
| 수동 운전 | 푸시버튼 — 누르는 동안만 ON(자기유지는 래더가 수행). 누르면 자동 시연 해제 |
| 자동 시연 | `▶ 자동 시연` 토글 — 라인별 운전 시퀀스 자동 수행 |
| 생산속도/불량률/수요 | 슬라이더(지원 라인) — KPI에 실측 반영 |
| 고장 주입 | 빨간 버튼 토글 — §5 효과, 재클릭=복구 |
| 카메라 | 휠=줌(0.55~3×), 드래그=이동, 더블클릭=초기화 |
| 기기 상세 | 3D 기기 라벨 클릭 — 신호·주소·전류·가동 통계(1s 갱신) |
| 내보내기 | 알람 패널 `CSV ↓`(이벤트 이력) / `리포트 ↓`(KPI·운전데이터·검증 md) |
| 서버 코사임 | API 서버 서빙 시 토글 — 10ms 정밀 스캔 + `트레이스 리플레이 ↻`(JSON 다운로드) |
| 엔지니어 상세 | 하단 접기 — 생성 ST 코드·사전 검증 타임라인 |
