# 안전 경계 고지 (필독)

> 실무 조사([`research/real-world-ladder-practices.md`](research/real-world-ladder-practices.md) §4)에서
> 가장 중요하게 확인된 현실 체크. 이 문서는 도구 사용 전 반드시 숙지한다.

## 핵심

**본 도구의 검증은 안전을 보장하지 않는다.**
이중코일 기계적 제거 · Z3 인터락 증명 · 도달성 검사는 **논리(logic) 보조**이며,
기능 안전(functional safety)과는 별개다.

## 왜 중요한가

- 표준 PLC 래더의 **소프트웨어 인터락은 안전 시스템이 아니다.** PLC 스캔에 의존하므로
  CPU 손상·버그·변조 시 실패한다. 하드웨어 인터락(배선·기계 링크)은 PLC가 죽어도 유효하다.
- **E-stop은 PLC가 읽고 판단하는 것이 아니라**, 안전릴레이를 통해 모터 접촉기로 전원을
  직접 차단(하드와이어)해야 한다. 크러시·절단·감전이 가능한 설비에 PLC-only E-stop은 안전하지 않다.
- PLC의 E-stop 인지는 **HMI 표시·알람 래치 용도**일 뿐, 실제 정지 경로는 전기기계적이고
  스캔 사이클과 독립적이어야 한다.

## 따라야 할 표준

| 표준 | 내용 |
|---|---|
| ISO 13849 | 안전 Performance Level (PL a~e), 기계적 안전. Cat 3/4=이중채널+교차감시 |
| IEC 62061 | 기계류 SIL 1~3, 전기/전자/프로그래머블 안전 |
| IEC 60204-1 | 기계 전기장비 안전 |
| NFPA 79 | 산업기계 전기 표준 (E-stop = self-latching 접점) |

- 정지 카테고리: **Cat 0**(즉시 전원차단), **Cat 1**(제어 정지 후 차단). E-stop은 Cat 0/1만 허용.
- PL d ≈ SIL 2, PL e ≈ SIL 3.

## 도구 사용 시 원칙

1. 생성된 래더/ST는 **유자격 엔지니어가 반드시 검토·검증**한다.
2. **SIL2+ 안전로직을 이 도구로 자율 생성하지 않는다.** (IEC 61508은 복잡 ML에 부적합 — 인증 공백)
3. 인터록 검증 통과 = "논리적으로 동시 ON 불가" 일 뿐, **안전 인증이 아니다.**
4. 안전 기능은 항상 래더 밖 **하드웨어 안전회로**로 이중화한다.

> 참고: 상용 코파일럿(Siemens Engineering Copilot, Rockwell FactoryTalk Design Studio Copilot)도
> 전부 "유자격 엔지니어 검증이 필요한 어시스턴트"로 포지셔닝하며 안전로직 자율생성을 하지 않는다.
