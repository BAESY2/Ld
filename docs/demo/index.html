자연어 → 검증된 PLC 래더 · 결정론 엔진

입으로 공장을 설계한다

PLC를 몰라도 평문으로 기획하면 — 결정론 엔진이 안전 검증된 래더를 만들고, 가상 PLC에서 1:1로 테스트합니다. Claude Code가 코드를, 우리는 공장을.

예시를 눌러보세요 — 실제 엔진이 생성·검증·가상가동한 결과로 즉시 이동합니다.
32 결정론 레시피 771 테스트 그린 이중코일 0 · 인터록 Z3 증명 OpenPLC 차분검증 bit-for-bit 5개 벤더 출력 — LS XGK/XGI·OMRON·Mitsubishi + Siemens SCL 래더 로직에 LLM 0%

왜 아직 아무도 못 했나 — 안전과 결정론

LLM은 확률(환각). 공장은 1과 0의 결정론. 1mm·1접점 오작동이 수억 설비·인명 사고로. 그래서 "자연어→즉시 기계"는 허상이었습니다. 우리는 LLM을 래더 로직에서 분리합니다.

① 자연어 기획한국어 평문 그대로. 슬롯 정규화 + 혼동어 교정(nlmatch)
② LLM — 의도 추출만파라미터·의도만 뽑음. 로직 생성은 금지(범위 밖이면 거절)
③ 상태머신 명세StateMachineSpec. IEC 61131-3 표준 타입만(BOOL/INT/TIME…)
④ 결정론 합성32개 검증 템플릿에서 ST 합성. 확률·환각 0%
⑤ ST + 래더 변환IEC ST → 래더 JSON. LS XGK 디바이스 주소 자동 할당(P/M/T/C/D)
⑥ 정형 검증Z3 SMT로 인터록 동시출력 UNSAT 증명·이중코일 0·도달성
⑦ 디지털트윈스캔 시뮬레이터 가상가동. 실제 OpenPLC와 bit-for-bit 차분검증
⑧ 안전 커널검증 통과분만 게이트 통과. 미검증 로직은 기계로 못 나감
⑨ 실기 제어LS XGT 프로토콜·Modbus로 PLC 직결. 5개 벤더 덱 출력
현장 가동모터·컨베이어·세차기… 아래 데모에서 직접 움직여 보세요 ↓

초록 단계 = LLM 손 안 댐(결정론) · 파랑 = LLM(추출만) · 노랑 = 실기. 환각이 기계에 닿지 못하는 구조가 우리의 해자.

지금 바로 체험 — 말로 만든 설계가 진짜로 움직인다
