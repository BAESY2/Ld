GitHub 링크를 열기만 하면 브라우저 안에서 진짜 엔진(WebAssembly)이 실행된다.

최초 1회 런타임(~10MB) 로딩 후 입력창에 한국어를 치면 즉시 설계가 뜬다.

```bash
pip install -e ".[web]"
uvicorn app.server:app --port 8000      # → http://localhost:8000/
```

### C. 명령행 / API
```bash
curl -X POST localhost:8000/api/compile \

  -d '{"text":"저수위 되면 펌프 켜고 고수위 되면 펌프 꺼"}'
```

## 2. 한국어 작성법 — 문법 패턴

엔진은 **조건 → 동작** 구조(한국어 SOV)를 이해한다. 자유 문장도 되지만 아래 패턴이
가장 정확하다.

### 기본형
- `<기기> <동작>` — 예: `펌프 켜`, `모터 돌려`, `밸브 열어`
- `<조건>면 <기기> <동작>` — 예: `저수위 되면 펌프 켜`

### 조건 종류
| 종류 | 예 | 컴파일 결과 |
|---|---|---|
| 버튼 | `버튼 누르면`, `시작 누르면` | START/STOP 트리거 |
| 수위 | `저수위 되면`, `고수위 되면` | LO_LS / HI_LS + 탱크 자동 합성 |
| 아날로그 | `압력 5바 넘으면`, `온도 200도 되면` | 비교기(PT/TT 계기) |
| 계수 | `부품 10개 차면`, `병 24개 차면` | 카운터(CTU) |
| 고장/검사 | `고장 나면`, `불량 나면`(비전 NG) | FAULT / NG_SENSOR |

### 동작 종류
`켜/꺼`, `돌려/멈춰`, `열어/닫아`, `배출해`, `울려`, (정지)`세워` 등. 무주어 동작
(`정지 누르면 멈춰`)은 **직전 기기**로 자동 해소된다.

### 복합문 (대규모)
조건-동작 절을 여러 개 이어 붙이면 한 프로그램으로 합성된다:
```
저수위 되면 펌프 켜고 고수위 되면 펌프 끄고 압력 5바 넘으면 밸브 닫고
부품 10개 차면 로봇 켜고 불량 나면 배출하고 고장 나면 사이렌 울려
```
→ 6개 서브시스템이 한 번에 컴파일·검증된다.

### 수량·단위
- 개수: `10개`, `24개` · 시간: `3초 후` · 동력: `5.5킬로와트`, `7.5kW`(소수 OK)
- 인스턴스: `1번 모터`, `펌프1`, `게이트A` → 고유 심볼(MOTOR1/PUMP1/GATEA)

## 3. 표준 제어 회로 카탈로그

각 항목은 **한국어 한 줄 → 검증된 회로 + 형식 증명**으로 나온다.

| 회로 | 입력 예 | 보장되는 안전 속성 |
|---|---|---|
| 자기유지 기동/정지 | `버튼 누르면 모터 돌고 정지 누르면 멈춰` | 정지 우선(1스캔 내 정지) |
| 수위 히스테리시스 | `저수위 되면 펌프 켜고 고수위 되면 펌프 꺼` | 채터링 방지 밴드 |
| **스타-델타 기동** | `5.5킬로와트 모터 스타델타로 기동해` | **Y⊥Δ 동시투입 금지 증명** + 산식 선정 |
| **정역 운전** | `정회전 돌리고 역회전 돌리고 동시에 안 되게` | **FWD⊥REV 동시투입 금지 증명** |
| **펌프 교번** | `펌프 두 대 교대로 운전해` | **1호기⊥2호기 증명** + 기동마다 교대 |
| 순차 시퀀서 | `모터 돌리고 다음 펌프 켜고 다음 밸브 열어` | **단계 one-hot 증명**(동시활성 ≤1) |
| 카운터 배출 | `부품 10개 차면 배출` | 프리셋 도달 트리거 |
| 상호배제 인터락 | `히터 켜고 쿨러 켜는데 동시에 안 켜지게` | **동시구동 금지 증명** |
| 보틀링 라인 | `컨베이어 돌리고 다음 충전기 켜고 다음 캡핑기 켜고 다음 배출해` | 라인 시퀀스 + 3D 연동 |

## 4. 화면 구성과 기능

| 탭/패널 | 설명 |
|---|---|
| **공장 도면 (P&ID)** | 계기 버블(PT/TT/HS)·기기 심볼·배관·신호선·표제란. 기기 클릭=상세. 휠/드래그/핀치 줌. |
| **제어반** | 동력 단선도(3Φ R/S/T → MCCB → INV → MC → EOCR → 부하) + AC220V 제어회로. |
| **3D 가동** | PBR 공장(바닥·I빔·가드레일·캐비닛) + 기기 모델 + 배관 흐름 + 통전 케이블. 자동 시연. |
| **래더** | XG5000 풍. 통전 글로우, 타이머 ET/PT·카운터 CV/PV 라이브. 줌(−/+/맞춤). |
| **ST 코드** | 합성된 IEC 61131-3 Structured Text. |
| **가상 PLC** | 입력 스위치 토글 → 실시간 스캔. **자동 시연**(열면 스스로 가동). |
| **운전 데이터** | 기기별 RUN/STOP·가동시간·기동횟수·전류(A)·부하율(%). |
| **기기 상세** | 태그·PLC 주소·부품 명세(BOM)·연결 + **정밀 테스트 버튼**. |
| **에러코드 진단** | 4사 101건 검색(코드/증상/LED). |

## 5. 정밀 테스트 읽는 법

도면에서 기기 클릭 → **정밀 테스트 ▶** (또는 운전 데이터의 *전체 정밀 리포트*):

| 검사 | 통과 의미 |
|---|---|
| 기동 전수탐색 | 입력 2ⁿ 전 조합 중 켜지는 조합을 빠짐없이 확인(도달 가능) |
| 자기유지 일치 | ST 구조와 실제 거동(입력 해제 후 유지)이 일치 |
| 정지 우선 | 기동+정지 동시 인가 시 1스캔 내 정지 |
| 인터락 퍼징 | 무작위 입력 폭격에도 증명된 짝 동시 ON 0회 |
| 타이머 정밀 | 첫 ON 시각이 프리셋과 ±1스캔 |
| 택트(단계 도달) | 단계별 첫 ON 시각 + 사이클 타임 + UPH(시간당 생산량) |
| 교차엔진 대조 | JS↔Python 시뮬 트레이스 비트 일치(서버판) |

## 6. 내보내기

기기 상세/내보내기 카드에서:
- **PLCopen XML** — OpenPLC Editor·CODESYS 임포트용 표준 XML.
- **LS XGK IL** — 벤더 명령어(IL) 텍스트.
- **설계 JSON** — ST·래더·설비·주소·검증 리포트 전체 페이로드.

## 7. 에러코드 진단

에러코드 카드에 코드·증상·LED명을 입력:
- 코드: `6706`(미쓰비시 인덱스 범위초과), `6105`(워치독)
- 증상/LED: `배터리`, `SF`(지멘스 시스템폴트), `통신`, `BF`
- 결과: 원인 + **실무 조치 절차** + 근거 출처 링크.

전체 자료집: [`docs/ERRORCODES.md`](ERRORCODES.md) (LS·미쓰비시·지멘스·오므론+공통 101건).

## 8. 거절될 때 (왜 confident=False)

엔진은 **모르면 정직하게 보류한다**(거짓 래더를 만들지 않는다). 보류 사유:
- **범위 밖**: PID·서보·모션·통신(Modbus)·HMI·잡담 → 컴파일 대상 아님.
- **의미 부적합**: `온도 올려`, `탱크 켜`(측정/용기는 구동 대상 아님).
- **이해 불확실**: 미등록 어휘·끊긴 종결 → 커버리지 하락.

이때 화면은 *이해 내용*과 *미해결 절*을 표시한다 — 무엇을 못 알아들었는지 보인다.

## 9. FAQ

**Q. API 키가 필요한가?** 아니다. 이해·합성·검증·시뮬·내보내기 전부 키 불필요(결정론).

**Q. 실제 PLC에 바로 넣나?** PLCopen XML/IL 로 내보내 벤더 IDE에서 임포트한다.
본 도구는 **설계 보조이며 안전 인증이 아니다** — 현장 투입 전 자격 엔지니어 검토 필수.

**Q. 3D/시뮬은 실물인가?** 아니다. **디지털 트윈 시뮬레이션**이다. 단, 이해·합성·검증·
정밀 테스트는 실제 엔진이 수행하고 결과는 형식 증명·교차검증된다.

**Q. 왜 한국어만?** 결정론 한국어 문법 엔진이 차별점이다(타 언어는 로드맵).

**Q. 안전 속성은 정말 증명되나?** Z3 SMT 솔버의 k-귀납으로 인터락·상호배제를 증명한다.
검증기 건전성은 false-proof 0 · miss 0 으로 측정된다([백서](WHITEPAPER.md) §4).
