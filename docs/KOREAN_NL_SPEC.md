## 4. 부정·조건 처리 (현재 미지원 → 잘못 매칭)
- 규칙: `_NEGATION_TERMS = ("안 ","못 ","말고","아니","금지","지 마","면 안")`. 부정어 + 상위2개가 혼동쌍이면 **confident=False + 구별질문**(의미 역전은 절대 시도 안 함, 자신있게 틀리지 않기).
- `동시투입금지/동시구동금지/동시에 켜지면 안` = 인터락 레시피(fwd_rev/conveyor_divert/star_delta) **양(+)의 부스트 토큰**.
- "정지 누르기 전까지 계속" → motor_start_stop("까지"가 temporal 처럼 보여도 on_delay 발화 금지).

## 5. 슬롯 추출 정규화 (P0 — 영향 최대)
현재 `_SEC_RE=(\d+(?:\.\d+)?)\s*초`, `_COUNT_RE=(\d+)\s*(?:개|번|회|ea)` 만 → 대부분 놓침.

**시간 → 초(int):** 모든 시간 토큰 합산.
```python
_UNIT_TO_SEC = {"시간":3600,"시":3600,"hr":3600,"h":3600,"분":60,"min":60,"m":60,"초":1,"sec":1,"s":1,"ms":0.001}
_TIME_TOKEN = re.compile(r"(\d+(?:\.\d+)?)\s*(시간|시|분|초|sec|s|min|m|ms)", re.I)
# 1시간30분=5400, 5분30초=330, 0.5초→clamp lo=1, 500ms→0.5→clamp, 5~10초→5(하한)
```
**한글 수사:** 한1 두2 세3 네4 다섯5 …열10 스물20 서른30 / 일이삼사오육칠팔구십. 단일 토큰 1~10·20·30 우선(11~19 복합은 TODO).
**개수:** 단위 확장 `개|개수|갯수|매|장|본|회|번|사이클|ea|pcs|pc`; 한글수사 `(한|두|…|열)\s*(개|번|회|매|장|본)`; 범위 `5~10개`→하한.
**단계별 시퀀서(P0):** 시간 토큰을 **순서대로** 수집해 k번째 → k번째 time_sec 필드(car_wash/heat_treat/plating/batch/traffic). 지금은 전부 첫 숫자로 붕괴됨.

## 6. 토크나이저 — 조사/어미 분리 (P1, rag._tokenize)
≥3글자 한글 토큰의 말미 조사/어미 제거: `-를/-을/-가/-이/-은/-는/-로/-으로/-에/-에서/-면/-하면/-하고/-해줘/-시켜/-게/-도록/-한다/-니다`. 남는 어간 ≥2글자일 때만, **안/못/말고는 절대 제거 금지**. BM25 정확매치율↑(현재는 느린 substring 경로 의존). rag 공유이므로 instruction-RAG 회귀 확인.

## 7. 라벨 코퍼스 (→ tests/ 픽스처)
연구가 60줄 `("문장","recipe_id",{슬롯})` 제공(모호행은 `_confident:False`). 비모호행=top1 일치+슬롯 포함, 안전어행=confident False+safety_warning. 적용 시 `tests/test_nl_corpus.py`로 박는다(CLAUDE 규칙4).

## 8. 적용 우선순위
P0 시간/개수 정규화 + 단계별 시퀀서 → P1 조사분리 + 키워드보강 → P2 혼동쌍6 + 부정가드 → P3 초generic 토큰 약가중. 전부 결정론·키 불필요·기존 analyze/extract_slots 흐름.
