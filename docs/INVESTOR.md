# 투자자 브리프 — 정직한 한 페이지 (연구 프리뷰)

> 과장하지 않는다. 못 하는 것은 못 한다고 적는다. 모든 수치는 `python scripts/capability_report.py` 로 재현된다.

## 한 줄
**자연어(한국어) 산업 제어 지시 → 형식 검증된 PLC 래더.** LLM 없이 결정론으로 이해·합성하고,
못 하는 것은 정직하게 거절하며, 만든 것은 기계 증명한다.

## 푸는 난제 (= 해자)
제어 로직을 짤 전문가는 희소하고, 틀린 코드는 설비파손·인명사고다. 그래서 LLM 환각으로는
*안전하게* 자동화할 수 없다. **우리 베팅: 검증기가 해자다.** 생성기는 누구나 좋아진다 —
방어 가능한 자산은 *무엇이 생성되든 안전을 기계증명하고, 못 고치면 거절하며, 반례로 수리하는*
검증+수리 루프다. **모델이 강해질수록 이 안전 게이트의 가치는 커진다.**

## 증거 (재현 가능·교차검증된 실수치)
| 항목 | 수치 |
|---|---|
| 한국어 이해(흔한 단일의도) 정확도 | 84% · **범위밖 침묵실패 0%** |
| 컴파일 + 적대(110건, PID·서보·통신 함정 포함) | **침묵실패 0** · confident 전부 검증통과·이중코일 0 |
| 생성 가능 공간(검증 통과한 서로 다른 프로그램) | **3,648개** (37 템플릿이 아니라 조합적) |
| 독립 백엔드 교차 동치(PySim↔XGK IL↔OpenPLC 트윈) | **100%** (트레이스 비트 일치) |
| 형식 증명 커버리지(선언 상호배제) | **71/71 = 100%** |
| 검증기 건전성@규모(무작위 1000+프로그램) | **false-proof 0 · miss 0** (양방향) |
| 자동 수리(고장 프로그램 → 재검증 통과) | **100%** |
| 회귀 테스트 / 골든 | **1,200+ passed · 21/21 · 이중코일 0 · 인터락 0** |

**헤드라인: "0% 침묵실패 — 모르는 건 거절, 만든 건 형식증명."** 비전문가에게 *틀린 래더를
자신있게 넘기지 않는* 것을 수치로 입증.

## 라이브 데모 (60초)
한국어 문장 → ① 이해 내용("조건: 저수위 도달 / 동작: 펌프 켬") → ② 검증된 ST/래더 →
③ 형식 안전 증명(동시 구동 불가 k-귀납) → ④ 디지털 트윈/OpenPLC 가동. `scripts/demo_e2e.py`.

## 정직한 한계 (숨기지 않는다)
- **트랙션 0** — 사용자·매출·파일럿 없음. 연구 프리뷰다.
- **범위가 좁다** — 불리언/시퀀스/기초 아날로그 비교까지. **PID·모션/서보·통신·HMI·대규모는 거절**(못 함).
- **검증은 인터락·시퀀스 상호배제 중심** — 타이밍 정량·아날로그 동역학의 완전성은 미완.
- **자체+교차검증** — 외부 독립 감사·실하드웨어 양산·안전인증(TÜV/SIL)은 범위 밖(조직 필요).

## 무엇을 찾는가
설계 파트너(LS/메카피온 생태계 공장·자동화 SI)와, *검증 보장 합성*을 깊게 밀 딥테크/안전 자본.
성장 라운드가 아니라, **난제를 증거로 푸는 단계**의 동행.
