# CAPABILITY — 정직한 통합 능력 스코어카드

> 정직이 유일한 해자다. 이 문서는 과장하지 않는다. 못 하는 건 못 한다고 적는다.

## 정체성 (한 줄)

**결정론 한국어 → 정형검증 PLC 컴파일러** (연구 프리뷰). 자연어(한국어) 제어 의도를
형태소→의도프레임으로 파싱해 IEC 61131-3 ST 로 *컴파일*하고, k-귀납 형식검증으로 안전을
증명하며, 못 하면 *정직하게 거절*한다. **LLM·환각 없음, API 키 불필요, 100% 결정론.**

이 스코어카드는 새 측정을 발명하지 않는다. 흩어진 벤치들(nl / realworld / compile /
adversarial / generativity / cross-backend / repair / soundness)의 핵심 함수를 *import 호출*해
한 화면으로 모은 것이다. 각 줄은 수치와 함께 *정직한 한계*를 단다.

## 대표 출력 (캡처)

`python scripts/capability_report.py` 의 대표 출력 (건전성 기본 규모 n=80):

```
==============================================================================
정직한 통합 능력 스코어카드 — 결정론 한국어→정형검증 PLC 컴파일러
(연구 프리뷰 · 키 불필요 · 100% 결정론 · 측정 없는 주장 금지)
==============================================================================
▸ 이해(NL in-template)
    정확도 84% (21/25) · 21개밖 침묵실패 0% (0/21)
    한계: 교과서형 단일의도에 한함. 보류 4건은 '맞을 수 있었으나 확신 못함'(거짓 매칭 대신 기권).
▸ 실세계(messy KO)
    인식 77% · 커버리지 40% · 침묵실패 2.1% (산출물 verify 100%)
    한계: 다중 서브시스템 *전체의도* 캡처는 구조적 0%(단일 레시피 한계). 정량/관계는 디지털 근사. messy 경로(BM25)는 침묵실패가 0 이 아니다(키워드 표면 겹침).
▸ 컴파일(코퍼스 52)
    confident 35/52 · verify 100% · 이중코일0 100% · 침묵실패 0
    한계: 아날로그 PID/모션/통신/잡담은 컴파일 대상 아님(거절). 미등록 어휘·끊긴 종결은 coverage 하락으로 기권 — 조용히 틀리진 않는다.
▸ 적대벤치(110)
    confident 65건 전부 verify·이중코일0 · 범위밖 45건 정직거절 100% · 침묵실패 0
    한계: in-vocab 명사가 섞여도 PID/서보/토크/통신 등 *제어 클래스* 밖이면 거절. literal 의미검사는 없다 — coverage 게이트에 의존한다.
▸ 생성성(combinatorial)
    무작위 조합 2400건 → 검증통과 *distinct* 1840개 (이중코일0)
    한계: 유한 어휘(조건10×동작14)의 조합 — 어휘 밖 의미(아날로그 PID 등)는 못 늘린다. 레시피가 아니라 원시어휘를 늘려야 범위가 넓어진다.
▸ 교차백엔드 동치
    PySim <-> XGK IL 21/21 (100%) · PySim <-> OpenPLC(트윈) 21/21 (100%)
    한계: 불리언 코퍼스만(아날로그 비교기 제외). OpenPLC 트윈은 오프라인 시뮬백 재생이라 정의상 PySim 과 일치 — 실기 OPENPLC_HOST 연결은 별도.
▸ 수리 효능(M4)
    고장 12건 → 수리·재검증 통과 12 (100%) · 잔존실패 0
    한계: 인터락 위반·이중코일 등 *국소* 결함만 건전 수리. 구조적 결함(one-hot 등)은 정직 거절(수리 대상 아님).
▸ 검증기 건전성(n=80)
    쌍: false-proof 0 · miss 0 · 증명확인 75 · 탐지 120/120 | 그룹: false-proof 0 · miss 0 · 증명확인 8 · 탐지 30/30
    한계: 여기선 작은 규모(random=80) — soundness_study.run_full 은 기본 1000개. 인터락(상호배제) 속성에 한함(타이밍·아날로그 의미론은 확대 중).
------------------------------------------------------------------------------
핵심 안전 불변(테스트가 단정):
    컴파일 침묵실패=0 · 적대 침묵실패=0 · confident 전부 안전(verify+이중코일0)=True
    건전성(n=80): 쌍 false-proof=0·miss=0 · 그룹 false-proof=0·miss=0
    → 모든 불변 성립: True
------------------------------------------------------------------------------
할 수 있는 것:
    결정론(키 없음) 한국어→정형검증 PLC 컴파일러로서: 흔한 단일의도를 검증된 ST/래더로 컴파일(이중코일0·인터락0), 모르는 것은 정직 거절(컴파일·적대벤치 침묵실패 0), 유한 어휘를 조합적으로(수천 distinct) 생성, 깨진 프로그램을 건전 수리, 산출물을 독립 백엔드 3종에서 교차검증(트레이스 100% 일치), 인터락 안전을 형식증명(건전성 false-proof=0·miss=0).
못 하는 것(정직):
    아날로그 PID/모션/서보/토크·통신(Modbus)·HMI 는 컴파일 대상이 아니다(거절). 다중 서브시스템의 *전체의도* 자동합성은 구조적 0%(단일 레시피 한계). messy(BM25) 경로는 침묵실패가 0 이 아니다. literal 의미검사 없이 coverage 게이트에 의존. 안전 인증(TÜV/SIL)·실하드웨어 양산은 범위 밖(조직 필요). 연구 프리뷰다 — 숨기지 않는다.
==============================================================================
```

> 위 수치는 캡처 시점의 대표값이다. 결정론이므로 같은 코드·코퍼스에서 재실행이 비트 동일하다
> (건전성의 무작위 부분도 시드 고정). 코퍼스/어휘가 늘면 수치는 갱신된다.

## 강점 (측정으로 뒷받침)

- **거짓 합성 0 (침묵실패 0)**: 컴파일 코퍼스(52)와 적대·대규모 코퍼스(110, in-vocab 함정
  포함) 모두에서 *범위밖을 자신있게 컴파일한 건수 = 0*. 모르는 것은 거절한다.
- **confident 산출물 전부 안전**: confident 컴파일은 100% verify 통과 + 이중코일 0
  (synth→verify 게이트가 보증).
- **조합적 생성성**: 고정 레시피가 아니다. 유한 어휘(조건 10 × 동작·기기 14)의 무작위 조합
  2400건에서 검증 통과한 *서로 다른* 프로그램 1840개. 어휘를 늘리면 표현 범위가 넓어진다.
- **독립 교차검증**: 같은 산출물을 PySim·XGK IL 인터프리터·OpenPLC 트윈 3종에 돌려 출력
  트레이스 100% 일치 — 단일 시뮬레이터 자가채점을 넘어선다.
- **건전 자동수리**: 인터락 위반·이중코일로 깨진 프로그램 12건을 자동 수리해 전부 재검증 통과.
- **검증기 건전성(해자)**: 무작위 + 실레시피 + 적대 뮤턴트에 걸쳐 k-귀납이 *증명한* 쌍·그룹을
  실행 오라클이 단 한 번도 반증 못함 → **false-proof = 0, miss = 0** (쌍·그룹 모두).

## 한계 (정직 — 못 하는 것)

- **아날로그·모션·통신은 컴파일 대상이 아니다**: PID/PI 루프, 서보·토크·위치 모션,
  통신(Modbus), HMI 트렌드는 거절한다(불리언 명세 밖).
- **다중 서브시스템 전체의도 = 0%**: 현 파이프라인은 요청당 단일 레시피만 고른다. 여러
  서브시스템을 요구하면 한 조각밖에 못 잡는다(개별 합성/LLM 설계 필요).
- **messy(BM25) 경로 침묵실패 ≠ 0**: 거친 실세계 한국어에서 키워드 표면이 겹치면 자신있게
  틀릴 위험이 남는다(스코어카드의 `실세계` 항목이 이 격차를 그대로 보고한다).
- **literal 의미검사 없음**: 범위밖 거절은 *coverage 게이트*(미등록 어휘→확신 강등)에
  의존한다 — 의미 수준의 거절 로직이 아니다.
- **건전성 속성 범위**: 현재 인터락(상호배제) 속성에 한정. 타이밍·아날로그 의미론은 확대 중.
- **조직이 필요한 것**: 안전 인증(TÜV/SIL), 실하드웨어 양산·시장 유통은 범위 밖이다.

## 재현법

```bash
source .venv/bin/activate
python scripts/capability_report.py                 # 사람용 스코어카드
python scripts/capability_report.py --json          # 기계용 JSON
python scripts/capability_report.py --soundness-n 1000   # 건전성 규모 상향(무거움)
```

검증(게이트):

```bash
ruff check app tests scripts
mypy app
pytest -q tests/test_capability_report.py
```

리포트는 핵심 안전 불변(컴파일·적대 침묵실패 0, confident 전부 안전, 건전성
false-proof/miss 0)이 깨지면 종료코드 1 을 반환한다(CI 게이트). 같은 불변을
`tests/test_capability_report.py` 가 단정한다.

## 구성 (어디서 무엇을 호출하나)

| 항목 | 출처 함수 |
| --- | --- |
| 이해(NL in-template) | `app.bench.nl_accuracy.score_keyless` + `benchmarks.nl_bench_corpus.BENCH` |
| 실세계(messy KO) | `scripts.run_realworld_bench.run` |
| 컴파일(코퍼스 52) | `scripts.compile_bench.run` / `total_silent_failures` / `all_confident_safe` |
| 적대벤치(110) | `scripts.compile_adversarial_bench.run` (동) |
| 생성성 | `scripts.generativity_bench.run` |
| 교차백엔드 동치 | `scripts.cross_backend_bench.run_bench` |
| 수리 효능 | `scripts.repair_bench.run` |
| 검증기 건전성 | `scripts.soundness_study.run_full` (작은 n) |

모든 호출은 *서브프로세스가 아니라 함수 import* 라 빠르고 결정론적이다. 무거운 건전성은
작은 규모(기본 n=80, 테스트 n=24)로 돌리되, false-proof/miss=0 은 규모 불변이다.
