## LS ELECTRIC (XGT/XGK/XGB)

### 통신

#### `CNET_NO_RESP` — Cnet(시리얼) 통신 무응답/타임아웃

**심각도**: 🟡 경고 · **시리즈**: XGB/XGT Cnet

- **원인**: 국번·보레이트·패리티 불일치, RS-485 A/B 극성 반전, 종단저항 누락.
- **조치(실무)**: 양단 파라미터(국번/속도/패리티/정지비트)를 표로 만들어 1:1 대조. 485 라인은 A-A/B-B 결선·말단 종단저항(120Ω) 확인. 프레임 모니터 기능으로 요청/응답 프레임을 캡처하면 어느 쪽이 침묵하는지 즉시 갈린다.
- **근거/출처**: [XGK 초급 교재](https://sol.ls-electric.com/uploads/document/16572866006190/XGK+초급(V21).pdf)
- **키워드**: Cnet, RS485, 통신, 타임아웃, 국번

#### `COMM_ERR` — XGB 통신 오류

**심각도**: 🟡 경고 · **시리즈**: XGB

- **원인**: 내장 시리얼 또는 Cnet 통신 포트의 응답 불량.
- **조치(실무)**: 통신 파라미터(보레이트, 프로토콜) 재확인, 케이블 및 종단저항 점검.
- **키워드**: XGB, communication, serial, Cnet, LS ELECTRIC

#### `FENET_IP_DUP` — FEnet(이더넷) 통신 두절 — IP 충돌/링크 다운

**심각도**: 🟡 경고 · **시리즈**: XGT FEnet

- **원인**: 동일 네트워크에 IP 중복, 스위치 포트 불량, 케이블 단선.
- **조치(실무)**: 모듈 LINK/ACT LED 로 물리 링크부터 확인 → arp/ping 으로 IP 중복 검사 (중복 시 한쪽 변경). 채널 점유 한계(동시 접속 수) 초과도 두절처럼 보이므로 접속 클라이언트 수를 확인.
- **근거/출처**: [XGK 초급 교재](https://sol.ls-electric.com/uploads/document/16572866006190/XGK+초급(V21).pdf)
- **키워드**: FEnet, 이더넷, IP 충돌, LINK

### I/O·모듈

#### `IO_CONF_ERR` — XGK I/O 구성 오류

**심각도**: 🔴 정지급 · **시리즈**: XGK

- **원인**: XGK 기저부에 장착된 모듈이 XG5000 프로젝트 I/O 파라미터와 불일치.
- **조치(실무)**: XG5000 I/O 파라미터를 실제 장착 모듈에 맞게 수정 후 재다운로드.
- **키워드**: XGK, I/O, configuration, LS ELECTRIC, XG5000

#### `IO_DETACH` — 운전 중 모듈 착탈 감지 — CPU 정지

**심각도**: 🔴 정지급 · **시리즈**: XGT 공통

- **원인**: 운전 중 I/O 모듈이 빠졌거나(진동·베이스 록 풀림) 접점 산화로 착탈로 오인식.
- **조치(실무)**: 모듈 록 레버·베이스 고정 나사 점검 후 재장착. 진동 심한 설비는 베이스 고정 보강. 재발 시 해당 슬롯 커넥터 청소(접점 세정제) 후 이력 클리어.
- **근거/출처**: [XGK 초급 교재](https://sol.ls-electric.com/uploads/document/16572866006190/XGK+초급(V21).pdf)
- **키워드**: 모듈 착탈, 탈락, 베이스, 정지

#### `IO_PARAM_MISMATCH` — I/O 파라미터 불일치 — 장착 모듈과 설정 모듈이 다름

**심각도**: 🔴 정지급 · **시리즈**: XGK

- **원인**: 슬롯에 장착된 모듈 종류가 I/O 파라미터 설정과 다르거나, 모듈 교체 후 파라미터를 갱신하지 않음.
- **조치(실무)**: XG5000 [I/O 파라미터]에서 슬롯별 설정과 실장 모듈을 대조. 모듈을 바꿨다면 파라미터 재작성 후 쓰기. 베이스 증설 케이블 접촉 불량도 동일 증상을 내므로 케이블 재삽입 확인.
- **근거/출처**: [XGK 초급 교재](https://sol.ls-electric.com/uploads/document/16572866006190/XGK+초급(V21).pdf)
- **키워드**: I/O, 파라미터, 모듈 불일치, 슬롯

### 전원/배터리

#### `BAT-LED` — BAT LED 점등 — 백업 배터리 전압 저하

**심각도**: 🟡 경고 · **시리즈**: XGT 공통

- **원인**: CPU 백업 배터리(리튬) 수명 종료 임박 — 정전 시 래치 데이터·시계 소실 위험.
- **조치(실무)**: 통전 상태에서 교체하면 데이터가 보존된다(정전 교체 시 래치 영역 소실 위험 — LS 공식 Q&A 절차). 교체 전 XG5000 으로 프로젝트/데이터 백업 필수. 기종별 배터리 형번은 공식 '기종별 배터리' 문서로 확인 후 정품 사용.
- **근거/출처**: [LS ssq Q&A: CPU 백업 배터리 교체방법](https://ssq.ls-electric.com/kr/ko/community/qna/document/25825)
- **키워드**: 배터리, BAT, 래치, 백업, 교체

#### `BATT_ERR` — XGK 배터리 이상

**심각도**: 🟡 경고 · **시리즈**: XGK

- **원인**: XGK CPU 모듈 배터리 전압 저하 또는 배터리 미장착.
- **조치(실무)**: 지정 규격의 배터리로 교체 후 메모리 데이터 재확인.
- **키워드**: XGK, battery, BATT, LS ELECTRIC, RAM backup

### 프로그램/연산

#### `PROG_ERR` — XGK 프로그램 오류

**심각도**: 🔴 정지급 · **시리즈**: XGK

- **원인**: 래더 프로그램 내 문법 오류, 잘못된 명령어 사용, 또는 END 명령 누락.
- **조치(실무)**: XG5000으로 오류 위치 확인 후 수정 및 재다운로드.
- **키워드**: XGK, program, error, ladder, LS ELECTRIC

### CPU/시스템

#### `CPU_ERR` — XGK CPU 하드웨어/자기진단 오류

**심각도**: 🔴 정지급 · **시리즈**: XGK

- **원인**: XGK CPU 모듈 내부 자기진단에서 하드웨어 이상 감지.
- **조치(실무)**: CPU 모듈 교체 및 LS ELECTRIC 기술지원 문의.
- **키워드**: XGK, CPU, error, self-diagnostic, LS ELECTRIC

#### `CPU_ERR` — XGB CPU 이상

**심각도**: 🔴 정지급 · **시리즈**: XGB

- **원인**: XGB CPU 자기진단 실패 또는 내부 하드웨어 불량.
- **조치(실무)**: CPU 모듈 전원 재투입 후 지속 시 모듈 교체.
- **키워드**: XGB, CPU, error, LS ELECTRIC

#### `ERR-LED-ON` — ERR LED 점등 — CPU 자기진단 에러(운전 정지급)

**심각도**: 🔴 정지급 · **시리즈**: XGT 공통

- **원인**: 자기진단에서 운전 불가 판정(워치독·모듈 구성 불일치·메모리 이상 등).
- **조치(실무)**: XG5000 접속 → [온라인]-[PLC 이력/에러 이력]에서 에러 코드·발생 시각을 먼저 읽는다(추측 금지). 이력의 코드로 원인 항목을 특정한 뒤 조치하고, 조치 후 에러 클리어→RUN 전환. 이력을 지우기 전에 반드시 캡처해 정비일지에 남길 것.
- **근거/출처**: [XGK 초급 교재(공식 공개 PDF)](https://sol.ls-electric.com/uploads/document/16572866006190/XGK+초급(V21).pdf)
- **키워드**: ERR, LED, 자기진단, 에러이력, XG5000

#### `ERR-LED-BLINK` — ERR LED 점멸 — 경에러(운전 지속 가능 경고)

**심각도**: 🟡 경고 · **시리즈**: XGT 공통

- **원인**: 운전은 지속되지만 주의가 필요한 상태(배터리 저하·퓨즈 단선·일부 모듈 경고).
- **조치(실무)**: 운전 중이라도 방치 금지. 에러 이력에서 경고 종류 확인 → 배터리/퓨즈류는 차기 PM(정기보전) 전이라도 즉시 교체 계획 수립. 경고 플래그(F 영역)를 HMI 에 알람으로 올려 두면 재발 시 즉시 인지 가능.
- **근거/출처**: [XGK 초급 교재](https://sol.ls-electric.com/uploads/document/16572866006190/XGK+초급(V21).pdf)
- **키워드**: ERR, 점멸, 경고, F플래그

### 워치독/스캔

#### `SCAN_WDT` — 스캔 워치독 초과 — 연산 지연으로 CPU 정지

**심각도**: 🔴 정지급 · **시리즈**: XGT 공통

- **원인**: FOR/CALL 중첩·대량 통신 처리로 1스캔이 워치독 설정을 초과.
- **조치(실무)**: XG5000 [기본 파라미터]의 워치독 시간과 실제 최대 스캔타임(특수 릴레이로 모니터)을 비교. 무거운 연산은 분할 실행하거나 고정주기 태스크로 이동. 워치독 값 상향은 원인 제거 후 최후 수단.
- **근거/출처**: [XGK 초급 교재](https://sol.ls-electric.com/uploads/document/16572866006190/XGK+초급(V21).pdf)
- **키워드**: 워치독, 스캔타임, WDT, 정지

#### `WDT_ERR` — XGK 스캔 워치독 초과

**심각도**: 🔴 정지급 · **시리즈**: XGK

- **원인**: XGK CPU 스캔 시간이 워치독 타이머 설정값을 초과.
- **조치(실무)**: 스캔 프로그램 최적화, XG5000 소프트웨어로 워치독 시간 재설정.
- **키워드**: XGK, watchdog, WDT, scan, LS ELECTRIC

#### `WDT_ERR` — XGB 워치독 타임아웃

**심각도**: 🔴 정지급 · **시리즈**: XGB

- **원인**: XGB 스캔 처리 시간이 워치독 설정을 초과.
- **조치(실무)**: 프로그램 처리 부하 감소, 워치독 설정 값 조정.
- **키워드**: XGB, watchdog, WDT, scan, LS ELECTRIC

## MITSUBISHI (MELSEC FX/Q)

### 통신

#### `6201-6205` — 통신(패리티/오버런/프레이밍) 에러 — 시리얼 품질 불량

**심각도**: 🟡 경고 · **시리즈**: FX

- **원인**: 보레이트/패리티 불일치, 케이블 길이 초과·노이즈, GND 루프.
- **조치(실무)**: 양측 통신 포맷을 표로 1:1 대조 → 차폐 케이블·실드 편측 접지 적용. 인버터 동력선과 통신선의 동일 덕트 배선이 단골 원인 — 배선 분리.
- **근거/출처**: [FX 트러블슈팅 정리](https://infoneva.com/en/knowledge/dealing-with-mitsubishi-fx-plc-error-codes)
- **키워드**: 통신, 패리티, 6201, 노이즈

#### `LINK_COMM_ERR` — iQ-R 네트워크 링크 통신 오류

**심각도**: 🟡 경고 · **시리즈**: iQ-R

- **원인**: CC-Link IE 또는 MELSECNET 링크 모듈의 통신 이상(케이블 단선, 국번 중복).
- **조치(실무)**: 네트워크 케이블·커넥터 점검, 국번 중복 제거, 링크 파라미터 재확인.
- **키워드**: iQ-R, CC-Link, MELSECNET, link, communication, Mitsubishi

### I/O·모듈

#### `IO_VERIFY_ERR` — iQ-R I/O 모듈 검증 오류

**심각도**: 🔴 정지급 · **시리즈**: iQ-R

- **원인**: 실제 장착된 I/O 모듈이 파라미터로 설정된 모듈 정보와 불일치.
- **조치(실무)**: GX Works3 파라미터를 실제 구성에 맞게 수정 후 재쓰기.
- **키워드**: iQ-R, I/O, verify, module, Mitsubishi, parameter

#### `M8060/D8060` — I/O 구성 에러 — 미실장 I/O 번호를 프로그램이 참조

**심각도**: 🟡 경고 · **시리즈**: FX

- **원인**: 증설 블록 탈락/미장착 상태에서 해당 X/Y 를 프로그램이 사용. D8060 값이 문제의 선두 I/O 번호를 가리킨다.
- **조치(실무)**: D8060 값으로 문제 디바이스 특정 → 증설 케이블·블록 전원(증설형은 별도 24V) 확인. 실제로 뺀 모듈이면 프로그램의 해당 X/Y 참조를 제거하거나 예비로 남길 거면 주석으로 명시.
- **근거/출처**: [FX3U 사용자 매뉴얼 에러표(공개 열람)](https://www.manualslib.com/manual/1234627/Mitsubishi-Fx3u-Series.html?page=261)
- **키워드**: I/O 구성, D8060, 증설, 미실장

### 전원/배터리

#### `BATT_ERR` — iQ-R 배터리 오류

**심각도**: 🟡 경고 · **시리즈**: iQ-R

- **원인**: iQ-R CPU 모듈 배터리 전압 저하.
- **조치(실무)**: 지정 배터리로 교체 후 프로그램·래치 데이터 재확인.
